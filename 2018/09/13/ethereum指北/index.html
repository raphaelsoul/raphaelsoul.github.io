以太坊介绍

以太坊是第二代区块链技术的代表。与第一代区块链技术相比，以太坊目前依旧沿用了PoW机制，PoS机制仍然还处于计划阶段。
同时以太坊通过实现一个虚拟机(EVM)，实现了智能合约。这也是第二代区块链技术创新性与革命性的特征。
至于dPoS与石墨烯技术为共识和底层技术的EOS，那就是另外一个话题了。
同时，与BitCoin相比区别又在于比特币从一开始就是通缩模型，总量是2100万个。大约4年出块奖励减半一次，直至最小单位不可分割，出块将不会得到奖励，
而以太坊15秒出一个块，并对分歧块（叔块）也提供奖励。自第二代区块链技术开始，鲜少再有通缩模型的区块链经济，
主要原因是，作为现实世界于虚拟世界的映射，我们需要区块链技术保留通胀的能力。

以太坊架构

以太坊目前有两个实现， 分别是c++和golang。从工程实现来说任何语言都可以实现，只是难度和工作量的问题。
作为一切的计算机科学世界的根基，C/C++自不用讨论， golang则因为goroutine的强大性能与特性，非常适合用来开发分布式系统。

  • cpp-ethereum
  • go-ethereum

总的来说，一个ethereum的项目实现，至少需要包含一下几部分：

  • crypto： 算法实现与白皮书中的交易验证机制
  • p2p: p2p网络广播
  • miner：挖矿程序
  • jsonrpc: 供dApp通过web3调用的jsonRPC服务
  • evm: 一个可以运行二进制智能合约的虚拟机
  • database: 高性能高可用的一个底层数据库。

顺带一提，ethereum选择了内嵌levelDB作为底层数据实现，EOS选择了内嵌MongoDB作为底层数据库实现

以太坊智能合约

这里只大概介绍一下过程，具体的相信一名合格的工程师都可以独立完成。

  1. 编写一个运算求和的智能合约，实现一个add(unit a, unit b)方法
  2. 将智能合约编译成字节码和ABI

    solc –abi –bin testContract.sol

  3. 解锁账户，部署智能合约，等待挖矿挖出
  4. 调用智能合约

    // 本地调用

    eth.contract(abi).at(address).add(1, 1)

    // 消费gas执行 并全网确认

    eth.contract(abi).at(address).add(1, 1, { from: eth.accounts[0], gas: 50000 })
