计算机原理

程序是如何执行的？

由于C/C++是一门底层语言， 我们需要先了解以下计算机运行原理和编译。
无论高级语言如JavaScript/Python，抑或是底层语言如C/C++/Java/Golang, 最底层的原理都是一致的。

暂时抛开计算机软硬件整体架构的演进，在物理层，硬件工程师们的所有计算机设计都可以简化为一种架构，即运算-存储，
作为核心的运算单元，最早由CPU承担，在应用程序越来越复杂的今天，硬件工程师们专门设计了GPU来负责图形的计算。

最早期的计算机可以认为就是个自动化的算盘，没有强有力的交互设备，非常蒸汽朋克。使用各种扳手开关输入指令，靠各种指示灯输出结果，那时候还没有ACSII编码标准，更没有UTF8等高级标准。因此只能进行数字运算
然后发展为纸带机，纸带并不是给机器看的，纸带是给人看的。因此人们需要先设置好机器让他能够用一定的标准来看懂纸带。因此纸带更像是最早的编程语言。

以计算A+B为例，指令序列和原始数据A和B将会先被放在寄存器中(往往是RAM内存)。
指令序列最终被CPU运算以与非门通电的形式进行，执行指令序列中规定的从哪个寄存器中取数据，执行什么样的指令处理数据，最终将数据放到哪里等。
1就是通电，0就是不通电。因此在出现基础物理学重大突破之前（例如量子计算机), 所有同原理的计算机理论能达到的最高运算速度，是受到光速限制的。具体涉及到更深入的微观物理学，就不拓展了。

而后，随着技术发展越来越快，这种运算-储存架构被进一步扩展， 变成了今天我们熟知的计算机硬件架构。

罗马不是一天建成的

计算机运算任务也不是一天变复杂的。随着硬件架构的发展，计算机运算的机制也有渐进式的演变。最终冯诺伊曼奠定了现代计算机的基本结构，即冯诺伊曼架构。

而CPU也因为各种历史原因产生了各种个样的架构，一般取决于纳米级的制程指令集，
前者一般是工业公司主导对CPU的架构进行设计，例如intel, AMD;
后者主要是学院派提出的指令操作标准，需要硬件层面的实现，例如我们熟知的x86, x86_64之外还有x66等比较冷僻的。
有些指令集术语历史过度产物， 有些现在依然在部分应用场景中发挥着作用。
但是至少，我们需要面对arm与intel平台上的x86指令集的32位和64位，共计四种组合。

指令集是CPU的抽象指令，非常晦涩难懂以至于需要intel出一本厚厚的手册来解释。因此我们有了汇编与opcode。更加底层，但是至少还有一点可读性的抽象代码。

编译原理

于是早期的变成语言基本都是编译型语言。所谓编译，就是早期纸带机制的演进，是从人类可读的代码，转成机器可执行的二进制代码的过程。
虽然有点类似，但是js打包从es6转成es5的过程，不叫编译(compile)，而是构建(build)。

一般的编译器分为前端和后端，处理的事情有些不同。

  • 预处理

    由前端完成，主要处理模块合并，行号追踪，注释移除，都是文本层面的处理

  • 词法分析

    由前端完成，分析关键词 构建AST语法抽象树

  • 语法分析

    由前端完成，遍历语法抽象树

  • 优化

    由前端完成，优化部分引用或操作

  • 生成汇编/opcode/二进制

    由后端完成，此时会针对不同的CPU平台的指令集生成不同的中间语言，最终变为二进制内容

  • 链接

    链接类库，一些系统级别的依赖，直接用或者放在某个位置等待动态链接。前者例如libcurl，后者例如php的一堆扩展

于是我们就这样得到了可执行的二进制文件。
至于非编译语言就有点不一样了。有各自的特性和实现，
基本都是在执行前把代码读入内存编译处理后执行，但这并不是导致动态类型语言性能满的原因， 动态语言性能慢的原因主要是因为类型推导需要消耗较多的性能。
因此现在的动态语言都在吸收兼容强类型语言的特点，通过声明变量类型绕过一定的类型推导。

what is C++？

我们总是能见到C和C++经常放在一起相提并论，那是因为C++最早成为C with class，是随着面向对象思想对现实世界的拟真再现的需求而出现的。
因此C++能够向下兼容C语言。某种意义上是C语言的扩展超集。

C++经过几十年的发展，也拥有了一定的标准，C98， C11，C14等等。

####
与动态语言不同，为了能更高效的利用内存，静态语言往往有多种变量类型

  • bool
  • char
  • int
  • float
  • double
  • void
  • wchar_t

同时 部分类型可以使用修饰符修饰 ，声明内存占用大小

  • signed
  • unsigned
  • short
  • long

我们可以用标准库std中的sizeof函数获取变量类型占用字节数，
用numberic_limits类获取这个变量类型的范围区间。

除了基本语类型外，还有枚举类型。甚至可以使用typedef关键字对基础类型进行别名。

数据结构

  • 数组

    和C语言中基本接近, 用来存储有序数据

  • 结构体

    用来存储结构化的数据，即kv结构数据，可以和typedef共同使用, golang继承了这个数据结构

  • 指针

    这是很多具体使用的小技巧，处理不好会造成内容溢出或者漏洞，是需要注意的地方。简单来说就是一个地址指向，保证两处使用同一个数据时，修改的是同一个内存地址的数据。

面向对象

C++最为一门面向对象的语言，自然有大多数面向对象特性，例如继承、重载、多态、接口等等。这一块不多做赘述，有面向对象基础很容易上手。
